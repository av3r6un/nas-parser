# Audit

## Назначение

`CatalogAudit` проверяет уже готовый `output/catalog.xlsx` после выполнения
pipeline.

Audit не участвует в ETL и ничего не исправляет. Его задача:

- быстро показать качество экспортированного каталога;
- выделить проблемные записи;
- подсказать точные номера строк, которые требуют проверки.

## Что проверяется

### Общая статистика

- общее количество строк каталога.

### SKU

- пустые `sku`;
- дублирующиеся `sku`.

Для дубликатов audit показывает:

- сам `sku`;
- все строки каталога, где он встречается.

### Name

- пустые наименования.

### Category

- статистика по категориям.

Audit не валидирует словарь категорий, он только считает их количество.

### Price

- пустые значения;
- отрицательные значения.

### Quantity

- пустые значения;
- отрицательные значения;
- значения в виде Excel-формулы, начинающиеся с `=`.

### Shape

- количество пустых значений.

### Fixation

- количество пустых значений.

### Cut

- количество пустых значений.

## Формат вывода

`summary()` состоит из двух частей.

### 1. Статистическая шапка

Всегда выводится:

- rows;
- counters по основным типам проблем;
- category counts.

### 2. Детализация проблем

Показывается только если реальные проблемы найдены.

Сейчас детализация есть для:

- duplicate SKU;
- empty SKU;
- empty price;
- empty quantity.

Во всех случаях используются реальные номера строк Excel-каталога, включая
смещение из-за заголовка.

## CLI

Запуск:

```bash
uv run python -m nas_parser.audit output/catalog.xlsx
python -m nas_parser.audit output/catalog.xlsx
```

Если путь к файлу не передан:

- печатается usage;
- команда завершается с ненулевым кодом.

Если файл отсутствует:

- печатается понятное сообщение;
- команда завершается с ненулевым кодом.
