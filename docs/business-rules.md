# Бизнес-правила

## SKU

### 12cut / 16cut

- итоговый `sku` читается из Excel;
- если `sku` уже присутствует в `ProductRecord`, `ProductEnricher` его не
  пересобирает.

### K9

- входной артикул отсутствует;
- `sku` строится автоматически в `ProductEnricher`.

Формат:

```text
K9/Color/Shape/Size/ColorCode
```

Пример:

```text
K9/Crystal/Drop/7x12/001
```

## Name

### Cut

- имя строится из цвета;
- результат используется как display name в каталоге.

### K9

- имя строится из цвета и формы.

## Category

Категория определяется по `fixation`.

- `hot` -> `Стразы горячей фиксации`
- `non` -> `Стразы холодной фиксации`
- `sew` -> `Пришивные стразы`

## Color code

- если цвет найден в reference-файле, используется код цвета;
- если код не найден, в export остается исходное название цвета.

## Size normalization

### K9

- `10*14` -> `10x14`
- `10MM` -> `10MM`

Разрешено только преобразование `*` в `x`.

Не меняются:

- регистр;
- `MM`;
- числовое содержимое;
- любые другие символы.

### Cut

- размер хранится так, как пришел из соответствующего формата.

## Shape

- `shape` заполняется только для K9;
- для cut-записей `shape=None`.

## Cut

- `cut` заполняется только для `12cut` и `16cut`;
- для K9 `cut=None`.

## Fixation

- для hotfix-файлов используется `hot`;
- для non-hotfix-файлов используется `non`;
- для пришивных используется `sew`.

## Export

Exporter не знает происхождение полей.

Он получает уже готовый `ProductRecord` и только отображает:

- `sku`
- `name`
- `price`
- `quantity`
- `category`
- `color`
- `size`
- `shape`
- `fixation`
- `cut`
