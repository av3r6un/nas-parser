## Production polish

В production-polish фазе архитектура уже считалась зрелой, поэтому
предпочтение отдавалось только минимальным локальным изменениям.

### Готовый артикул для cut

- `CutParser` начал читать `Артикул` из Excel;
- `ProductEnricher` перестал пересобирать `sku`, если он уже заполнен.

### Mix support

- для `size == "mix"` в cut-файлах инвертировано расположение колонок
  `Артикул` и `Код цвета`;
- правило полностью инкапсулировано внутри `CutParser`.

### K9 Premium

- `K9Parser` начал поддерживать `K9_premium.xlsx`;
- поддержка реализована по `stem.startswith("k9")`.

### Нормализация размеров K9

- размеры вида `10*14` нормализуются в `10x14`;
- размеры вида `10MM` остаются без изменений.

### Audit

- добавлен отдельный `CatalogAudit` для готового `catalog.xlsx`;
- audit получил CLI;
- audit научился показывать проблемные строки каталога.

### RuntimeWarning fix

- `nas_parser.__init__` перестал импортировать CLI entrypoint;
- запуск `uv run python -m nas_parser.main` больше не вызывает
  `RuntimeWarning`.

## Вывод

К моменту подготовки `v1.0.0` проект представляет собой:

- завершенный production ETL;
- отдельный аудит качества результата;
- покрытую тестами кодовую базу;
- набор зафиксированных правил, которые важно сохранять при дальнейшем
  развитии.
